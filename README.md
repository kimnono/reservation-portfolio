# Timekeeper Frontend

사내 자원 예약 서비스를 가정한 프론트엔드 포트폴리오입니다.  
사용자 예약 흐름과 관리자 백오피스 흐름을 분리해, 예약 생성부터 승인/반려와 자원 관리까지 한 프로젝트 안에서 보여주는 것을 목표로 했습니다.

## 무엇을 보여주는 프로젝트인가

- 사용자 영역과 관리자 영역을 분리한 예약 UX
- 예약 생성, 상세 조회, 취소, 승인/반려 흐름
- 회의실, 좌석, 장비를 포함한 자원 관리 구조
- `React Query`, `React Hook Form`, `Zod`, `Zustand`를 조합한 상태 관리 방식
- 실제 백엔드 없이도 흐름을 검증할 수 있도록 mock API와 인메모리 DB 기반으로 구성한 구조

## 주요 화면

- 공개 영역
  - 개요 화면
  - 자원 타임테이블
  - 예약 생성
  - 내 예약 목록
  - 예약 상세
- 관리자 영역
  - 대시보드
  - 예약 목록 / 필터 / 정렬 / 상태 변경
  - 자원 관리
  - 자원 스케줄 보드
  - 사용자 요약 화면

## 기술 선택 의도

- `Next.js 16`
  - App Router 기반으로 공개 영역과 관리자 영역의 진입 흐름을 구조적으로 분리하기 위해 사용
- `React 19`
  - 최신 클라이언트 컴포넌트 패턴과 화면 전환 흐름을 반영하기 위해 사용
- `Tailwind CSS 4`
  - 빠른 화면 구성과 반복 스타일 정리를 위해 사용
- `@tanstack/react-query` v5
  - 서버 상태 캐시와 갱신 흐름을 분리하기 위해 사용
- `React Hook Form + Zod`
  - 폼 상태와 검증 로직을 명확하게 분리하기 위해 사용
- `Zustand`
  - 관리자 필터 UI, 자원 편집 모달 같은 국소 상태를 가볍게 관리하기 위해 사용

## 현재 범위

- 백엔드 없이 mock API + repository + 인메모리 DB 구조로 동작
- 인메모리 DB 기반이므로 개발 서버 재시작 시 데이터 상태는 초기화될 수 있음
- 권한별 진입 경로 분리
- 예약 충돌 처리와 상태 변경 반영
- 사용자 자원 화면에서 자원별 일간 타임테이블 확인 가능
- 빈 30분 슬롯 클릭 후 예약 생성 화면으로 바로 이동 가능
- 타입체크 / 린트 / 프로덕션 빌드 기준 검증 가능

## 구조

- `app`
  - 라우트, 레이아웃, 페이지 엔트리
- `features`
  - 예약, 인증, 관리자 기능 단위 코드
- `shared`
  - 공통 UI, 포맷 유틸, query key
- `mocks`
  - 인메모리 DB, repository, mock API 연결
- `styles`
  - 반복되는 Tailwind 스타일 상수
